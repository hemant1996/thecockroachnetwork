6 images. Koi company ise band nahi kar sakti — kyun, aur trust kahan se aata hai.

Six pictures. Why no company can shut this down, and where trust comes from.

01 / 06

Aap khud
sign karte ho. You sign
yourself.

No signup, no email, no OTP. Pehli baar app khulne par aapka phone ek secret key banata hai — usi phone mein rehti hai, kisi server par nahi jaati.

Har report uss key se sign hoti hai. Identity = aapki public key. Account hai hi nahi — to password leak hone ka sawaal nahi.

No signup, no email, no OTP. The first time you open the app, your phone makes a secret key — it stays on the device and never leaves.

Every report is signed by that key. Identity = your public key. No account = no password to lose.

Secret key tumhare phone se kabhi nahi nikalti. Sirf signed event nikalta hai.
The secret key never leaves the phone. Only the signed event leaves.
02 / 06

Ek baar likho —
kayi jagah pahunche. Write once,
lands everywhere.

Publish dabaate hi aapki report ek saath sab Pehredaaron ke paas chali jaati hai. Abhi 3 chal rahe hain (Mumbai, Singapore, Render) — teeno ke paas ek-ek copy.

Twitter/Insta = ek company ka ek server. Yahan = 3 alag log, 3 alag servers.

Hit publish and your report goes to every Pehredaar at once. 3 running today (Mumbai, Singapore, Render) — each gets a copy.

Twitter/Insta = one company, one server. Here = 3 different people, 3 different servers.

Har Pehredaar ko wahi event milta hai. 3 alag log. 3 alag desh / cloud.
Each Pehredaar receives the same event. 3 different people. 3 different countries / clouds.
03 / 06

Pehredaar khud aapas mein
baat karte hain. Pehredaars talk to
each other.

Aap Mumbai par, koi aur Singapore par publish karta hai — phir bhi sab kuch sab ko dikhta hai. Pehredaar aapas mein lagaatar reports pass karte hain. Signature check karke — koi bhi fake event daal nahi sakta.

No silos. Sab Pehredaaron par sab kuch hota hai.

You publish to Mumbai, someone else to Singapore — you both see both. Pehredaars pass reports between themselves constantly. Each event's signature is checked first, so no one can fake anything.

No silos. Every Pehredaar carries everything.

Har Pehredaar baaki sabko jaanta hai. Koi master nahi hai.
Every Pehredaar knows every other Pehredaar. There is no master.
04 / 06

Ek Pehredaar band?
Network zinda hai. One Pehredaar dies?
Network lives.

Mumbai Pehredaar band ho gaya — server crash ho, government notice aaye, ya operator chhod kar chala jaaye. Koi farak nahi. Baaki 2 ke paas saara data hai (§03). Phone apne aap unse padhne lagta hai.

Koi ek insaan ya company is network ko band nahi kar sakti. Jitne zyada Pehredaar, utna mazboot — aap bhi ban sakte ho, 3 minute.

Mumbai Pehredaar goes down — server crash, government notice, or the operator walks away. Doesn't matter. The other 2 still hold everything (§03). Your phone auto-reads from whoever's alive.

No single person or company can shut this down. More Pehredaars = more resilient — you can become one too, 3 minutes.

1 Pehredaar down ≠ network down. Saara data baaki ke paas hai.
1 Pehredaar down ≠ network down. The others still hold all the data.
05 / 06

Pehredaar koi bhi
ban sakta hai. Anyone can be
a Pehredaar.

Abhi ke 3 Pehredaar humare hain — yeh permanent nahi. Koi bhi apna chalu kar sakta hai.

Render.com par ek button → 3 minute mein live (free, koi credit card nahi). Raspberry Pi, purana laptop, $5 ka VPS — kuch bhi chalega. Code CC0 public domain, permission nahi chahiye. Naya Pehredaar apne aap baaki ke saath jud jaata hai.

Jitne zyada Pehredaar — utna sach.

The 3 current Pehredaars happen to be ours — that's not permanent. Anyone can run their own.

One button on Render.com → live in 3 minutes (free, no credit card). Or a Raspberry Pi, an old laptop, a $5 VPS — anything works. Code is CC0 public domain, no permission needed. A new Pehredaar auto-joins the rest.

More Pehredaars = more truth.

CC0 = koi license, koi permission nahi. Bas Pehredaar chalu karo, network mein judne mein 1 second.
CC0 = no license, no permission. Start your Pehredaar; it joins the network in 1 second.
06 / 06

Naye Pehredaar share se
milte hain. Torrent jaise. New Pehredaars come in
through shares. Like torrents.

Naya Pehredaar aapke phone tak pahunchega kaise? Koi admin "approve" nahi karta. Toh kaise?

Jab koi report ka link share karta hai, us link ke andar Pehredaar ka address chhupa hota hai. Aapka phone link kholte hi seedha us address par check karta hai — sach mein cockroach Pehredaar hai? Pass ho gaya → apne aap aapki list mein add ho jaata hai, "via share" tag ke saath.

Bilkul torrent jaisa. Magnet link → swarm. Share-link → Pehredaar network.

Trust kahan se aaya? Uss insaan se jisne aapko link share kiya. WhatsApp dost, family group, jis Twitter handle ko follow karte ho — wahi log decide karte hain. Community ne banaya.

How does a new Pehredaar reach your phone? No admin "approves" it. So how?

When someone shares a report link, the Pehredaar's address rides inside the link. Your phone opens the link and goes straight to that address to check — is this actually a cockroach Pehredaar? If yes → auto-added to your list, tagged "via share".

Exactly like torrents. Magnet link → swarm. Share-link → Pehredaar network.

Where does trust come from? From the person who shared the link with you. Your WhatsApp friend, your family group, the Twitter handle you follow — they decide. The community built it.

Magnet link → torrent swarm. Share-link → cockroach Pehredaar network. Wahi tareeqa, alag content.
Magnet link → torrent swarm. Share-link → cockroach Pehredaar network. Same idea, different content.

Ab cockroach ban jao. Now be a cockroach.

Theory padhne se nahi, use karne se dikhta hai. Likhiye — har Pehredaar tak pahunchega, hamesha ke liye signed.

Not theory — try it. Write something. Every Pehredaar gets it, signed forever.
